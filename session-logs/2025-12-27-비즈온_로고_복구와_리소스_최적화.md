# 비즈온 로고 복구와 리소스 처리의 최적화

_(부제: 2025-12-27, 누락된 정적 자원을 복구하고 스크롤 반응형 로고 시스템을 구축하다)_

## 발단: 보이지 않는 브랜드 아이덴티티

레오는 비즈온 마케팅 페이지에서 로고 이미지와 배경 영상이 정상적으로 출력되지 않는 현상을 발견했다. 서브도메인을 활용한 멀티테넌시 환경에서 특정 정적 자원들이 404 에러를 일으키며 디자인의 완성도를 저해하고 있었다. 브랜드의 핵심인 로고가 부재한 상황은 사용자 경험 측면에서 즉각적인 해결이 필요한 과제였다.

## 전개: 자원 가로채기 문제의 확인

안티그래비티는 원인 파악을 위해 미들웨어의 라우팅 로직을 검토했다. 서브도메인 기반의 리라이트(Rewrite) 시스템이 이미지와 영상 같은 정적 파일의 요청 경로까지 가로채어 발생한 문제였다. 안티그래비티는 `middleware.ts`의 정적 파일 탐지 로직을 보강하고, `BizonMarketing` 컴포넌트 내의 이미지 태그들을 Next.js 표준 방식으로 교체하며 자원 복구를 시작했다.

## 위기: 왜곡된 비율과 의도치 않은 렌더링

초기 복구 작업 이후 로고 이미지가 원본 비율을 잃고 찌그러지거나, 하단의 보조 텍스트를 제거하기 위한 클리핑이 어긋나는 문제가 관찰되었다. 또한, 페이지 라우팅 우선순위의 충돌로 인해 공들인 마케팅 페이지 대신 기본 템플릿이 렌더링되는 현상이 추가로 보고되었다. 시스템의 복잡도가 증가함에 따라 여러 기술적 변수가 중첩되어 발생한 난관이었다.

## 절정: 필터 기반의 테마 전환과 라우팅 정렬

레오는 로고가 스크롤 위치에 따라 화이트와 컬러 테마로 자연스럽게 전환되어야 한다는 구체적인 피드백을 주었다. 안티그래비티는 CSS의 `clip-path`와 `filter`를 결합하여 별도의 이미지 수정 없이 로고 하단을 제거하고 색상을 제어하는 코드를 작성했다. 또한 `SitePage`의 라우팅 레이어를 재정의하여 프리미엄 페이지가 최우선으로 노출되도록 강제했다. 스크롤에 반응하여 로고의 오렌지 포인트가 드러나는 순간 기능은 완성되었다.

## 결말: 인터랙티브 브랜드 시스템의 완성

이번 작업을 통해 비즈온 페이지는 안정적인 자원 로딩 환경과 더불어 사용자의 행동에 섬세하게 반응하는 인터랙션을 갖추게 되었다. 레오와 안티그래비티는 인프라 수준의 경로 제어부터 말단의 UI 스타일링까지 아우르는 정교한 협업 과정을 거쳤다. 이로써 플랫폼은 브랜드별 고유 아이덴티티를 더 선명하게 전달할 수 있는 기반을 확보했다.

### 🛠 기술적 각주 (Technical Summary)

- **수정된 파일 (Modified Files)**:
  - `middleware.ts`: 정적 자원 탐지 정규표현식 보강 및 시스템 경로 예외 처리.
  - `app/sites/[site]/bizon-marketing.tsx`: `brightness-0 invert` 필터와 `clip-path`를 이용한 로고 구현.
  - `app/sites/[site]/page.tsx`: 하드코딩된 특정 페이지에 대한 라우팅 우선순위 최상단 배치.
- **주요 기술 (Key Tech Stack)**:
  - **CSS Filter Interactivity**: 스크롤 상태값에 따른 실시간 이미지 필터 제어.
  - **Next.js Rewrite Logic**: 다중 도메인 환경에서의 정적 자원 보존 체계.
- **핵심 기능 (Key Features)**:
  - **Scroll-aware Logo Theme**: 배경색 변화에 따라 로고의 가독성을 보장하는 지능형 스위칭.
  - **Priority Bypass Routing**: 서비스 중요도에 따른 조건부 렌더링 시스템.
