# 디자인 시스템과 폰트 일관성 확보

_(부제: 2025-12-26, 브랜드 아이덴티티를 결정짓는 폰트 시스템을 구축하고 디자인 가이드를 문서화하다)_

## 발단: 디자인 파편화와 폰트 충돌

레오는 플랫폼 전체의 심미적 통일성을 위해 전용 폰트 시스템의 도입을 요구했다. 한글은 프리텐다드(Pretendard), 영문은 팝핀스(Poppins)를 사용하여 브랜드의 세련미를 극대화하고자 했다. 하지만 당시 프로젝트는 Next.js의 기본 폰트와 Tailwind의 스타일이 혼재되어 브라우저마다 각기 다른 가독성을 보이고 있었다.

## 전개: 조화로운 폰트 스택의 구성

안티그래비티는 전역 스타일 파일인 `globals.css`를 열어 두 폰트의 우선순위를 정의하기 시작했다. 한글과 영문이 각자의 영역에서 최적의 가독성을 발휘할 수 있도록 폰트 패밀리(Font Family) 순서를 정렬하고, 스크롤 애니메이션 라이브러리인 AOS를 모든 마케팅 페이지에 이식하여 시각적 리듬감을 부여했다.

## 위기: 가려진 스타일과 FOUT 현상

수정 후 예상치 못한 변수가 나타났다. 브라우저가 페이지를 로드할 때 폰트가 깜빡하며 교체되는 FOUT(Flash of Unstyled Text) 현상이 발생했고, 특정 컴포넌트에서는 시스템 폰트가 강제로 적용되지 않는 현상이 관찰되었다. Tailwind CSS의 기본 설정이 사용자 정의 스타일보다 높은 우선순위로 페이지를 장악하고 있었기 때문이었다.

## 절정: CSS 레이어의 재정의

안티그래비티는 문제의 원인이 스타일 로드 순서와 캐스케이드(Cascade) 우선순위에 있음을 파악했다. `@import "tailwindcss"` 이후에 위치하던 스타일들을 Tailwind의 `@layer base` 내부로 이동시키는 구조적 개선을 단행했다. 레오의 검토 하에 모든 폰트 선언에 강력한 강제력을 부여하는 로직을 적용하자, 비로소 모든 페이지에서 일관된 Poppins와 Pretendard의 이중주가 실현되었다.

## 결말: 재사용 가능한 디자인 시스템의 정립

안티그래비티와 레오는 이번 작업을 통해 확정된 폰트 크기, 간격, 컬러 등 모든 디자인 요소를 `design-system.md`라는 가이드북으로 집대성했다. 이는 향후 추가될 모든 새로운 페이지들에 동일한 디자인 언어를 즉시 이식할 수 있는 환경을 조성한 것이다. 단순히 폰트를 바꾼 것을 넘어, 플랫폼의 영구적인 디자인 주권을 확립한 의미 있는 진척이었다.

### 🛠 기술적 각주 (Technical Summary)

- **수정된 파일 (Modified Files)**:
  - `app/globals.css`: Tailwind v4 레이어 시스템 기반 폰트 스택 오버라이드.
  - `app/layout.tsx`: 기본 Geist 폰트 제거 및 전역 폰트 클래스 적용.
  - `hooks/use-aos.ts`: 스크롤 인지형 애니메이션 파라미터 최적화.
  - `design-system.md`: 플랫폼 표준 레이아웃 및 타이포그래피 가이드 문서화.
- **주요 기술 (Key Tech Stack)**:
  - **Pretendard & Poppins**: 다국어 환경에 최적화된 폰트 렌더링 체계.
  - **Tailwind CSS Cascade**: 레이어 기반의 스타일 우선순위 관리 기술.
- **핵심 기능 (Key Features)**:
  - **FOUT Minimization**: 언어별 폰트 사전 정의를 통한 렌더링 시각적 안정화.
  - **Standardized Typography**: PC와 모바일을 아우르는 반응형 타이포그래피 스케일 정립.
